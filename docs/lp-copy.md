## 2.5 いま起きている変化（対話 → 任せる）

リード: **AIは「対話する相手」から、「任せるパートナー」へ。**
サブ: 指示して答えをもらう使い方から、目的を渡して成果を任せる段階へ。この移行で問われるのは、AIの操作スキルだけではありません。

| | これまで | これから |
|---|---|---|
| 役割 | 対話の相手 | 任せるパートナー |
| 使い方 | 都度プロンプトで指示し、答えをもらう。成果は"使う人のスキル"頼み。 | 目的・背景を渡し、プロセスごと成果を任せる。AIが自律的に動く。 |

氷山の比喩:
- 見える部分: **AIの使い方**
- 見えない部分（差がつく土台）: **考え方・進め方・協働の型**

締め: **→ AIに「任せる」ほど、ビジネス推進の基礎スキルの土台を再構築することが必要になる。**

## 3. 課題提起（なぜ、組織でAIのリテラシー・熱量が上がらないのか）

リード: **「AIを使え」の号令だけでは、組織は変わらない。**

| # | 原因 | 結果 |
|---|---|---|
| 1 | 断片的な情報 | SNS等から入る、不確かで一貫性のない表層的なインプット |
| 2 | 一時的な詰め込み | 研修や動画で詰め込む → 一時的に満足してすぐ忘れる |
| 3 | 個々人バラバラのインプット | 議論や協働が起きず、組織的な実践に繋がる勢いが生まれない |

締め: **→ 知識が組織的な行動に繋がらず、「AI号令」で終わってしまう。**

## 4. ソリューション（成果創出 両輪スキル）

リード: **成果は「AIスキル」と「ビジネスOSスキル」の両輪で生まれる。**
サブ: 弊社独自の、よくある研修にはない体系。

### AIスキル ── AIで効率と品質を高め、業務を再構築する実装力
- 土台層: 目的・背景・問いを伝える / プロセス・課題分解（Goal・Context・Issue）
- 運用層: 対話・選択・磨く（Prompting）/ 業務定義・手順化 / データ統合・連携（RAG）
- 構築層: 組み立てる・繋ぐ・自律化する（Orchestration / Agentic AI / MCP）
- 変革層: 新規価値創出・プロセス再構築（BX / DX / CX）

### ビジネスOSスキル ── 成長と収益を生むビジネスの体系的土台
- 土台層: 収益構造の把握・設計（LTV / KPI / ファイナンス）/ 顧客理解と価値創出（WHO / WHAT）
- 顧客体験層: CX創出（4P / 広告 / LTV向上 / ブランド）/ 営業（インサイドセールス / CS）
- 推進層: 推進スキル（PDCA / 協働 / PM）/ 自走スキル（自律 / コミットメント）
- 変革層: デジタル変革（DX / データ / エンジニアリング）/ 事業創造（新規事業 / イノベーション）

## 5. サービス体験（3本柱）

見出し: **チーム全体でスキルアップ。学びっぱなしにしない、実践を生み出す体験。**

1. **圧倒的に続く学び** ── ポケットのスマホで、いつでもどこでも。業務を圧迫しない1日10分から。
2. **議論による共通言語化** ── 課題や行動が揃う定例議論を、弊社がまるっと司会・進行。
3. **実践につながる一手** ── 学んだエッセンスごとに、AIが実践アイデアを貴社に合わせて自動提案。

経営に有用なレポートで可視化: 学びの進捗 → 理解度（スコア82%）→ スキル状態（バランス診断）→ 課題・アイデア

---

## 6. アプリ独自の学習体験

見出し: **スキマ学習で続き、チームで学び合い、思考が揃う。独自開発のe-ラーニング。**

- **圧倒的な継続実績**: 「1日10分」「スキマ時間」「チリツモ学習」。物理的制約を極限まで排除。
- **品質の高い学習**: 能動性・双方向性・ストーリー性のあるチャット小説型。受動的な動画学習の弱点を克服。
- **復習性**: 操作性の高いクリップ、メモ記入。
- **思考を深めるクイズ**: 合否判定テスト・正解率の可視化。
- **アンケート機能（思考アウトプット型）**: 「アイデア」「課題」を自分／自社ごと化。回答すると同僚のアイデアが見られ、"答え合わせ"のように理解が深まる。
- **ランキング**: 進捗・学習時間・クリップ数で組織を活性化。
- **管理ダッシュボード**: 管理者はリアルタイムで学習・理解状況を把握。
